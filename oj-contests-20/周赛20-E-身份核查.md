# E - 身份核查

**难度**：★★★★★  
**考点**：哈希表应用（unordered_map）  
**GESP 级别**：5级

---

## 题目背景

数字王国的边境检查站每天需要核查大量过客的身份。王国系统存储了所有公民的 ID 号与姓名的对应关系，检查站需要快速响应每次查询：给定一个 ID，返回对应的姓名，或提示身份不存在。

## 题目描述

给定 $n$ 名公民的 ID 和姓名，再进行 $m$ 次查询。对每次查询，若 ID 存在，输出对应姓名；否则输出 `NOT_FOUND`。

## 输入格式

第一行一个正整数 $n$（$1 \le n \le 100000$）。

接下来 $n$ 行，每行两个值：

```
id name
```

$id$ 为正整数（$1 \le id \le 10^9$），$name$ 为不含空格的字符串（长度 $\le 20$）。保证所有 $id$ 互不相同。

第 $n+2$ 行一个正整数 $m$（$1 \le m \le 100000$）。

接下来 $m$ 行，每行一个正整数，为待查询的 $id$。

## 输出格式

共 $m$ 行，每行输出对应的姓名或 `NOT_FOUND`。

## 样例

**输入**

```
3
1001 Alice
1002 Bob
1003 Carol
3
1002
9999
1001
```

**输出**

```
Bob
NOT_FOUND
Alice
```

**说明**：查询 1002 对应 Bob，查询 9999 不存在输出 NOT_FOUND，查询 1001 对应 Alice。

## 提示

使用 `unordered_map<int, string>` 存储 ID 到姓名的映射，查询时间复杂度为 $O(1)$。

```cpp
unordered_map<int, string> mp;
// 录入
mp[id] = name;
// 查询
if (mp.count(query_id)) cout << mp[query_id] << "\n";
else cout << "NOT_FOUND\n";
```

若使用 `map<int, string>`（红黑树），每次查询为 $O(\log n)$，$n, m$ 均达 $10^5$ 时也能通过，但 `unordered_map` 更高效。
