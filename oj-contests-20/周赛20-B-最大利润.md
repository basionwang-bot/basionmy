# B - 最大利润

**难度**：★★☆☆☆  
**考点**：数组遍历，维护前缀最小值  
**GESP 级别**：2级

---

## 题目背景

数字王国的集市每天都会公布王国货币的兑换价格。一位商人想在集市上买入并卖出货币以获取最大利润。规则是：只能进行一次买入和一次卖出，且必须先买后卖（卖出时间严格晚于买入时间）。

## 题目描述

给定 $n$ 天的价格序列，求能获得的**最大利润**。若无法获利（价格单调不增），则输出 $0$。

## 输入格式

第一行一个正整数 $n$（$1 \le n \le 10^5$）。

第二行 $n$ 个正整数，第 $i$ 个表示第 $i$ 天的价格（$1 \le$ 价格 $\le 10^4$）。

## 输出格式

输出一个整数，表示最大利润。

## 样例

**输入 1**

```
6
7 1 5 3 6 4
```

**输出 1**

```
5
```

**说明**：第 2 天以价格 1 买入，第 5 天以价格 6 卖出，利润为 $6-1=5$，是最优方案。

---

**输入 2**

```
4
7 6 4 3
```

**输出 2**

```
0
```

**说明**：价格单调递减，任何买卖都会亏损，最大利润为 $0$。

## 提示

遍历价格序列，维护当前见过的**最低价格** `minPrice`，每天计算以当天价格卖出时的利润 `price - minPrice`，取最大值即为答案。

```cpp
int minPrice = prices[0], ans = 0;
for (int i = 1; i < n; i++) {
    ans = max(ans, prices[i] - minPrice);
    minPrice = min(minPrice, prices[i]);
}
```

注意更新 `minPrice` 在计算利润之后也可以，但顺序是：先用当前价格尝试卖出，再更新最小价格，确保买在卖之前。
