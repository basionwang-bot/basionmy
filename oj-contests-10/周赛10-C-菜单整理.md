# C - 菜单整理

**难度**：★★☆☆☆
**考点**：结构体、条件筛选、双关键字排序
**GESP 级别**：3级

---

## 题目背景

餐厅每月对菜品进行整理，剔除销量不达标的菜品，并把剩余菜品按销量从高到低排序，方便厨房和采购参考。若两道菜销量相同，按价格从低到高排（让性价比更高的排前）。

## 题目描述

给定 $n$ 道菜的信息（编号、价格、销量）和销量达标线 $X$。请筛选出 $\text{sales} \geq X$ 的菜品，按以下规则排序后输出：
- 主关键字：销量降序；
- 次关键字（销量相同时）：价格升序。

若没有任何菜品达标，输出 `None`。

## 输入格式

第一行两个整数 $n$（$1 \leq n \leq 100$）和 $X$（$0 \leq X \leq 10000$）。

接下来 $n$ 行，每行三个整数 $id$、$price$、$sales$，分别表示菜品编号、价格、销量（$1 \leq id \leq 9999$，$1 \leq price \leq 999$，$0 \leq sales \leq 10000$）。

## 输出格式

若存在达标菜品，每行一道，格式为 `id price sales`，按上述规则排序；否则输出 `None`。

## 样例

**输入**

```
5 50
101 25 80
102 30 50
103 18 100
104 22 45
105 35 60
```

**输出**

```
103 18 100
101 25 80
105 35 60
102 30 50
```

**说明**

| id | price | sales | 是否达标（sales≥50） |
|----|-------|-------|----------------------|
| 101 | 25 | 80 | ✓ |
| 102 | 30 | 50 | ✓ |
| 103 | 18 | 100 | ✓ |
| 104 | 22 | 45 | ✗ |
| 105 | 35 | 60 | ✓ |

达标 4 道菜，按销量降序：100 > 80 > 60 > 50。

## 提示

- 定义结构体 `Dish { int id, price, sales; }`；
- 先扫描一遍筛选出 `sales >= X` 的菜品；
- 用自定义比较函数排序：`sales` 不同时按 `sales` 降序，相同时按 `price` 升序。

```cpp
bool cmp(Dish a, Dish b) {
    if (a.sales != b.sales) return a.sales > b.sales;
    return a.price < b.price;
}
```
