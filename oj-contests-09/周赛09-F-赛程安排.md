# F - 赛程安排

**难度**：★★★☆☆
**考点**：贪心算法（区间调度）
**GESP 级别**：6级

---

## 题目背景

运动会只有一块主场地，需要在上面安排多个比赛项目。每个项目有固定的开始时间和结束时间，同一时刻场地只能举办一个项目（下一个项目的开始时间必须 $\geq$ 上一个项目的结束时间）。请问最多能安排多少个项目？

## 题目描述

给定 $n$ 个比赛项目，每个项目有开始时间 $s$ 和结束时间 $e$（$s < e$）。选择尽可能多的互不重叠的项目，输出最多能安排的项目数。

**不重叠条件**：若先安排项目 $A$（结束时间为 $e_A$），下一个项目 $B$ 的开始时间 $s_B \geq e_A$ 时才视为不重叠。

## 输入格式

第一行一个整数 $n$（$1 \leq n \leq 100$）。

接下来 $n$ 行，每行两个整数 $s$ 和 $e$（$0 \leq s < e \leq 1000$）。项目输入顺序不保证有序。

## 输出格式

一个整数，表示最多能安排的项目数。

## 样例

**输入**

```
5
1 3
2 5
4 6
5 7
6 9
```

**输出**

```
3
```

**说明**

将5个项目按**结束时间**排序：

| 项目 | 开始 | 结束 |
|------|------|------|
| A | 1 | 3 |
| B | 2 | 5 |
| C | 4 | 6 |
| D | 5 | 7 |
| E | 6 | 9 |

贪心选择过程：
- 选 A（结束时间最早），记录 `last=3`
- B 的开始时间 2 < 3，跳过
- C 的开始时间 4 ≥ 3，选 C，`last=6`
- D 的开始时间 5 < 6，跳过
- E 的开始时间 6 ≥ 6，选 E，`last=9`

共选 3 个：A、C、E。

## 提示

**贪心策略**：每次优先选择**结束时间最早**的项目，这样为后续项目留出最多时间，是区间调度的最优贪心策略。

**算法步骤**：
1. 按结束时间从小到大对所有项目排序；
2. 初始化 `last = -1`（上一个已选项目的结束时间），计数器 `cnt = 0`；
3. 遍历排好序的项目，若当前项目开始时间 `≥ last`，则选择该项目，更新 `last` 并 `cnt++`；
4. 输出 `cnt`。

```cpp
sort(events, events + n, cmp);  // 按结束时间升序
int cnt = 0, last = -1;
for (int i = 0; i < n; i++) {
    if (events[i].start >= last) {
        cnt++;
        last = events[i].end;
    }
}
```
