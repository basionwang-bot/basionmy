# F - 最优编码

**难度**：★★★★★  
**考点**：哈夫曼树（Huffman Tree），最小带权路径长度（WPL）  
**GESP 级别**：6级

---

## 题目背景

魔法学院的信息传输系统需要对字符进行最优编码，使得编码总长度最短。这正是哈夫曼编码的应用：根据各字符出现的频率，构建哈夫曼树，让出现频率高的字符获得更短的编码。

## 题目描述

给定 $n$ 个字符的频率，构建哈夫曼树，输出其**带权路径长度（WPL）**。

WPL 定义：每个字符的频率乘以它在哈夫曼树中的深度（从根开始，根深度为 0），所有字符求和。

哈夫曼树构建算法：
1. 将所有频率放入小顶堆；
2. 每次取出最小的两个值 $a$ 和 $b$，合并为 $a + b$，并将 $a + b$ 重新放入堆；
3. 重复步骤 2，直到堆中只剩一个元素；
4. 每次合并操作中，$a + b$ 就是该子树贡献给 WPL 的值，将所有合并值累加即为 WPL。

## 输入格式

第一行一个正整数 $n$（$2 \le n \le 1000$）。

第二行 $n$ 个正整数，表示各字符的频率（$1 \le$ 频率 $\le 10000$）。

## 输出格式

输出一个整数，表示哈夫曼树的带权路径长度（WPL）。

## 样例

**输入 1**

```
6
5 9 12 13 16 45
```

**输出 1**

```
224
```

**说明**：
- 第1次合并：$5 + 9 = 14$，WPL 累加 14；堆：{12, 13, 14, 16, 45}
- 第2次合并：$12 + 13 = 25$，WPL 累加 25；堆：{14, 16, 25, 45}
- 第3次合并：$14 + 16 = 30$，WPL 累加 30；堆：{25, 30, 45}
- 第4次合并：$25 + 30 = 55$，WPL 累加 55；堆：{45, 55}
- 第5次合并：$45 + 55 = 100$，WPL 累加 100；堆：{100}
- 总 WPL = $14 + 25 + 30 + 55 + 100 = 224$

---

**输入 2**

```
4
1 2 3 4
```

**输出 2**

```
19
```

**说明**：
- 第1次合并：$1 + 2 = 3$，WPL 累加 3；堆：{3, 3, 4}
- 第2次合并：$3 + 3 = 6$，WPL 累加 6；堆：{4, 6}
- 第3次合并：$4 + 6 = 10$，WPL 累加 10；堆：{10}
- 总 WPL = $3 + 6 + 10 = 19$

## 提示

使用 C++ 优先队列（小顶堆）实现：

```cpp
priority_queue<int, vector<int>, greater<int>> pq;
for (int i = 0; i < n; i++) { int x; cin >> x; pq.push(x); }
long long wpl = 0;
while (pq.size() > 1) {
    int a = pq.top(); pq.pop();
    int b = pq.top(); pq.pop();
    int merged = a + b;
    wpl += merged;
    pq.push(merged);
}
cout << wpl << "\n";
```

注意：WPL 可能较大，用 `long long` 存储（最坏情况：1000 个频率均为 10000，WPL 约为 $10^8$，`int` 不会溢出，但养成使用 `long long` 的习惯更安全）。
