## 测试点 1（样例1：经典哈夫曼例题）

**输入**
```
6
5 9 12 13 16 45
```
**预期输出**
```
224
```
**验证**：合并顺序 5+9=14, 12+13=25, 14+16=30, 25+30=55, 45+55=100；WPL = 14+25+30+55+100 = 224。

---

## 测试点 2（样例2：n=4）

**输入**
```
4
1 2 3 4
```
**预期输出**
```
19
```
**验证**：合并顺序 1+2=3, 3+3=6, 4+6=10；WPL = 3+6+10 = 19。

## 测试点 3（n=2，最小规模）

**输入**
```
2
1 1
```
**预期输出**
```
2
```
**验证**：只有一次合并 1+1=2，WPL = 2。两个字符各编码长度为 1，$1 \times 1 + 1 \times 1 = 2$。

**输入**

3

**验证**：合并 2+3=5，再合并 4+5=9；WPL = 5+9 = 14。

## 测试点 5（n=5，频率全相等）

**输入**
```
5
1 1 1 1 1
```
**预期输出**
```
12
```
**验证**：
- 合并 1+1=2，WPL+=2；堆：{1, 1, 1, 2}
- 合并 1+1=2，WPL+=2；堆：{1, 2, 2}
- 合并 1+2=3，WPL+=3；堆：{2, 3}
- 合并 2+3=5，WPL+=5；堆：{5}
- 总 WPL = 2+2+3+5 = 12

---

## 测试点 6（n=4，频率大小不一）

**输入**
```
4
7 5 2 4
```
**预期输出**
```
35
```
**验证**：
- 合并 2+4=6，WPL+=6；堆：{5, 6, 7}
- 合并 5+6=11，WPL+=11；堆：{7, 11}
- 合并 7+11=18，WPL+=18；堆：{18}
- 总 WPL = 6+11+18 = 35

## 答案（C++ 参考代码）

```cpp
#include <iostream>
#include <queue>
#include <vector>
using namespace std;
int main() {
    int n; cin >> n;
    priority_queue<int, vector<int>, greater<int>> pq;
    for (int i = 0; i < n; i++) {
        int x; cin >> x;
        pq.push(x);
    }
    long long wpl = 0;
    while (pq.size() > 1) {
        int a = pq.top(); pq.pop();
        int b = pq.top(); pq.pop();
        int merged = a + b;
        wpl += merged;
        pq.push(merged);
    }
    cout << wpl << "\n";
    return 0;
}
```

**关键点**：
1. `priority_queue<int, vector<int>, greater<int>>` 声明小顶堆，每次 `top()` 取最小值；
2. 每次取出两个最小值合并，将合并值放回堆，同时累加合并值到 WPL；
3. 循环直到堆中只剩一个元素（`pq.size() > 1`）；
4. 每次合并操作的代价（合并值）之和即为 WPL，这等价于所有叶节点频率乘以各自深度之和；
5. 用 `long long` 存储 WPL 防止潜在溢出。
