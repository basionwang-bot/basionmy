# F - 石子合并

**难度**：★★★★★  
**考点**：区间DP  
**GESP 级别**：8级

---

## 题目背景

侦探事务所庭院中有一排石子，需要将它们逐步合并成一堆。每次只能合并**相邻**的两堆石子，合并代价等于两堆石子的数量之和。求将所有石子合并为一堆的**最小总代价**。

## 题目描述

给定 $n$ 堆石子（一字排开），每次可以合并任意**相邻**的两堆，代价为这两堆石子总数。将所有石子合并为一堆，输出最小总代价。

## 输入格式

第一行一个正整数 $n$（$1 \le n \le 300$）。

第二行 $n$ 个正整数，表示每堆石子数量（$1 \le a_i \le 1000$）。

## 输出格式

输出一个整数，表示最小总合并代价。

## 样例

**输入 1**

```
4
3 4 5 2
```

**输出 1**

```
28
```

**说明（验证）**：
- 先合并第3和第4堆：$5+2=7$，得 $[3,4,7]$，代价 7
- 再合并第1和第2堆：$3+4=7$，得 $[7,7]$，代价 7
- 最后合并：$7+7=14$，代价 14
- 总代价 $7+7+14=28$（此方案即为最优）

---

**输入 2**

```
3
1 2 3
```

**输出 2**

```
9
```

**说明**：先合并 $[1,2]$ 代价 3，再合并 $[3,3]$ 代价 6，总 9；或先合并 $[2,3]$ 代价 5，再合并 $[1,5]$ 代价 6，总 11。最优为 9。

---

**输入 3**

```
1
42
```

**输出 3**

```
0
```

**说明**：只有一堆石子，无需合并，代价为 0。

## 提示

设 $dp[i][j]$ 为合并第 $i$ 堆到第 $j$ 堆的最小代价，$sum[i][j]$ 为这段的总石子数：

$$dp[i][j] = \min_{i \le k < j} (dp[i][k] + dp[k+1][j] + sum[i][j])$$

```cpp
// 预处理前缀和
for (int i = 1; i <= n; i++) pre[i] = pre[i-1] + a[i];
// sum(i,j) = pre[j] - pre[i-1]

// 区间DP：从短区间到长区间
for (int len = 2; len <= n; len++) {         // 区间长度
    for (int i = 1; i + len - 1 <= n; i++) { // 起点
        int j = i + len - 1;                  // 终点
        dp[i][j] = INT_MAX;
        for (int k = i; k < j; k++) {         // 分割点
            dp[i][j] = min(dp[i][j],
                dp[i][k] + dp[k+1][j] + pre[j] - pre[i-1]);
        }
    }
}
cout << dp[1][n];
```

时间复杂度 $O(n^3)$，$n=300$ 时约 $2.7 \times 10^7$，1秒内可通过。
