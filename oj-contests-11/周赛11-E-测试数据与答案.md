2 20

**输入**
```
2 2
```

```
2
```

## 测试点 3（区间内无质数）

**输入**
```
8 10
```
**预期输出**
```
None
```

**说明**：8、9、10 均为合数（8=2³，9=3²，10=2×5）。

---

## 测试点 4（l=1，1 不算质数）

**输入**
```
1 1
```
**预期输出**
```
None
```

**说明**：1 既不是质数也不是合数。

---

## 测试点 5（大数据：靠近上界）

**输入**
```
9970 9980
```
**预期输出**
```
9973
```

**说明**：9973 是不超过 10000 的较大质数之一；9971 = 13² × 59 等其余均为合数。

## 答案（C++ 参考代码）

```cpp
#include <iostream>
using namespace std;

const int MAXN = 10005;
bool composite[MAXN];

int main() {
    composite[0] = composite[1] = true;
    for (int i = 2; (long long)i * i < MAXN; i++) {
        if (!composite[i]) {
            for (int j = i * i; j < MAXN; j += i)
                composite[j] = true;
        }
    }

    int l, r;
    cin >> l >> r;

    bool found = false;
    for (int i = l; i <= r; i++) {
        if (!composite[i]) {
            if (found) cout << " ";
            cout << i;
            found = true;
        }
    }
    if (!found) cout << "None";
    cout << endl;

    return 0;
}
```

**关键点**：
1. 全局数组 `composite[]` 默认初始化为 0（false），无需手动清零；
2. 筛起点 `j = i * i` 是优化：小于 $i^2$ 的合数已被更小的质因子筛过；
3. 输出时第一个数前不加空格，之后每个数前都加；
4. `i * i` 可能溢出 int（$i$ 最多到 $\sqrt{10005} \approx 100$，本题不溢出，但用 `long long` 是更稳妥的写法）。
