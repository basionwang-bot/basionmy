# D - 战场地图分析

**难度**：★★★☆☆
**考点**：二维数组进阶、最值查找、行统计、矩阵顺时针旋转 90°

---

## 题目背景

勇者的队伍获得了一张 $n \times m$ 的战场地图，每个格子标注了该区域的战略价值（正数为有利地形，负数为危险地带，0 为中立）。队长需要从三个维度分析这张地图，并将地图旋转 90° 以适配新的战术屏幕。

## 题目描述

给定一张 $n \times m$ 的战场地图，请完成以下三项分析：

1. **最高价值格**：找出价值最高的格子坐标（行号和列号，均为 1-indexed）；若有多个格子价值相同且均为最高，输出行号最小的，行号相同则输出列号最小的。
2. **最优行**：找出各行价值之和最大的行号；并列时取行号最小的。
3. **旋转地图**：将地图**顺时针旋转 90°** 后输出（旋转后地图大小为 $m \times n$）。

**顺时针旋转 90° 规则**（0-indexed）：原矩阵第 $i$ 行第 $j$ 列的元素，旋转后位于新矩阵的第 $j$ 行第 $(n-1-i)$ 列。

## 输入格式

第一行两个整数 $n$ 和 $m$（$1 \leq n, m \leq 20$）。

接下来 $n$ 行，每行 $m$ 个整数（$-100 \leq$ 值 $\leq 100$），表示战场地图。

## 输出格式

- 第一行：最高价值格的行号和列号，用空格分隔。
- 第二行：最优行的行号。
- 第三行起：顺时针旋转 90° 后的地图（$m$ 行，每行 $n$ 个整数，用空格分隔）。

## 样例

**输入**

```
2 3
1 2 3
4 5 6
```

**输出**

```
2 3
2
4 1
5 2
6 3
```

**说明**

原地图：
```
1  2  3   ← 第1行，行和 = 6
4  5  6   ← 第2行，行和 = 15（最大）
```

- 最高价值格：6，位于第 2 行第 3 列 → 输出 `2 3`
- 最优行：第 2 行（行和 15）→ 输出 `2`
- 旋转后（3×2）：
  - 新第1行：原第1列从下到上 → 4, 1
  - 新第2行：原第2列从下到上 → 5, 2
  - 新第3行：原第3列从下到上 → 6, 3

## 提示

- 旋转公式：建立新数组 `b[m][n]`，令 `b[j][n-1-i] = a[i][j]`，然后输出 `b`。
- 遍历顺序：先行后列地遍历原矩阵，按公式填入新矩阵，最后二重循环输出新矩阵。
- 注意输出数字之间用空格分隔，每行末尾无多余空格。
