# F - 任务调度

**难度**：★★★★★  
**考点**：拓扑排序（BFS Kahn 算法）  
**GESP 级别**：7级

---

## 题目背景

商队在沙漠中需要完成一系列任务，某些任务必须在另一些任务完成后才能开始。请找出一种合法的任务执行顺序。

## 题目描述

有 $n$ 个任务（编号 $1$ 到 $n$）和 $m$ 条依赖关系，每条关系 $(a, b)$ 表示任务 $a$ 必须在任务 $b$ 之前完成（即 $a \to b$）。

输出一个合法的**拓扑序**（满足所有依赖关系的执行顺序）。若存在循环依赖（无法找到合法顺序），输出 `IMPOSSIBLE`。

若存在多个合法拓扑序，输出**字典序最小**的那个（即每次优先选编号最小的可执行任务）。

## 输入格式

第一行两个正整数 $n$、$m$（$1 \le n \le 500$，$0 \le m \le 5000$）。

接下来 $m$ 行，每行两个整数 $a$、$b$，表示 $a$ 必须在 $b$ 之前完成（$1 \le a, b \le n$，$a \ne b$）。

## 输出格式

若存在合法拓扑序，输出 $n$ 个整数（用空格分隔），表示字典序最小的拓扑序。

若不存在，输出 `IMPOSSIBLE`。

## 样例

**输入 1**

```
4 3
1 3
2 3
3 4
```

**输出 1**

```
1 2 3 4
```

**说明**：任务 1 和 2 无依赖（可先选编号小的 1），再执行 2，然后 3，最后 4。

---

**输入 2**

```
3 3
1 2
2 3
3 1
```

**输出 2**

```
IMPOSSIBLE
```

**说明**：1→2→3→1 形成环。

## 提示

使用 BFS Kahn 算法，配合**最小堆（优先队列）**保证字典序最小：

```cpp
priority_queue<int, vector<int>, greater<int>> pq; // 小顶堆
// 初始化：入度为0的所有节点入堆
// 每次取最小编号节点，减少其后继节点入度，入度变0则入堆
// 若最终处理了n个节点，则无环；否则有环
```
