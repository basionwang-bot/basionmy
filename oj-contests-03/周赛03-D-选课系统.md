# 周赛 03 - D 题：选课系统

**难度**：★★★☆☆  
**考点**：结构体、排序、查询

---

## 题目背景

新学期开始了，学校上线了一套简易的「选课系统」。  
每门课程都有课程编号、课程名称、学分和已选人数。  
学生在选课前，想先查看有哪些课程可以选，并按一定的规则排序。

小强受老师委托，来实现这个简化版的查询功能。

---

## 题目描述

系统中共有 n 门课程，每门课程包含以下信息：

- 课程编号（不含空格的字符串）；  
- 课程名称（不含空格的字符串）；  
- 学分（非负整数）；  
- 已选人数（非负整数）。

现在需要按如下规则对所有课程排序后输出：

1. 先按学分从高到低排序；  
2. 若学分相同，则按已选人数从高到低排序；  
3. 若学分和已选人数都相同，则按课程编号的字典序从小到大排序。

请使用结构体存储每门课程的信息。

---

## 输入格式

第一行一个整数 n（1 ≤ n ≤ 100）。  
接下来 n 行，每行依次包含：

`课程编号 课程名称 学分 已选人数`  

字段之间用一个空格分隔，课程编号和课程名称均为不含空格的字符串。

---

## 输出格式

按排序后的顺序输出 n 行，每行格式为：

`课程编号 课程名称 学分 已选人数`

---

## 样例

**输入**
```
3
CS101 DataStruct 4 80
CS102 Algo 3 100
CS201 OS 4 60
```

**输出**
```
CS101 DataStruct 4 80
CS201 OS 4 60
CS102 Algo 3 100
```

**样例解释**

- CS101 与 CS201 的学分都是 4，比 CS102 的 3 高，应排在前面；  
- 在两门 4 学分课程中，按已选人数从高到低：80 > 60，因此 CS101 在 CS201 前；  
- 学分和已选人数都相同的情况按课程编号字典序排序，这里没有用到。
