# C - 菜谱排行

**难度**：★★★☆☆  
**考点**：结构体 + 条件筛选 + 多关键字排序  
**GESP 级别**：3级

---

## 题目背景

美食厨房要做一期"招牌菜榜单"。每道菜都有名称、销量和评分。只有**评分达到 80 分及以上**的菜才有资格上榜，再按销量从高到低排名。

## 题目描述

给定 $n$ 道菜，每道菜有名称、销量、评分三项信息。

筛选出**评分 $\ge 80$** 的菜，按以下规则排序后输出菜名（每行一个）：

1. **销量从高到低**；
2. 销量相同时，按**菜名字典序从小到大**。

若没有任何一道菜评分达到 80，输出 `none`。

## 输入格式

第一行一个正整数 $n$（$1 \le n \le 1000$）。

接下来 $n$ 行，每行格式为：

```
菜名 销量 评分
```

菜名为不含空格的字符串，销量与评分均为整数（$0 \le$ 销量 $\le 10^6$，$0 \le$ 评分 $\le 100$）。

## 输出格式

按排序规则，每行输出一个上榜菜名；若无菜上榜，输出 `none`。

## 样例

**输入**

```
5
KungPao 120 90
MapoTofu 200 75
Dumpling 200 95
FriedRice 80 88
HotPot 150 95
```

**输出**

```
Dumpling
HotPot
KungPao
FriedRice
```

**说明**：MapoTofu 评分 75 < 80，被淘汰。其余四道按销量降序：Dumpling(200)、HotPot(150)、KungPao(120)、FriedRice(80)。

## 提示

```cpp
struct Dish { string name; int sales, rating; };

sort(v.begin(), v.end(), [](const Dish& a, const Dish& b){
    if (a.sales != b.sales) return a.sales > b.sales; // 销量降序
    return a.name < b.name;                            // 菜名升序
});
```
