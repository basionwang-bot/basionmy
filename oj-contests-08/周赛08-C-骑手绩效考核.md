# C - 骑手绩效考核

**难度**：★★☆☆☆
**考点**：结构体、条件筛选、多关键字排序
**GESP 级别**：3级

---

## 题目背景

外卖平台每月对骑手进行绩效考核，评定"优秀骑手"的标准为：**完成订单数 ≥ 20 且好评率 ≥ 90%**。满足条件的骑手按好评率从高到低排名，好评率相同则按完成订单数从多到少排名。

## 题目描述

给定 $n$ 名骑手的信息，筛选出符合优秀骑手标准的骑手，按上述规则排序后输出其编号。

若没有骑手符合条件，输出 `None`。

## 输入格式

第一行一个整数 $n$（$1 \leq n \leq 100$）。

接下来 $n$ 行，每行三个整数，分别表示骑手编号 $id$、完成订单数 $orders$、好评率 $rating$（$1 \leq id \leq 9999$，$0 \leq orders \leq 200$，$0 \leq rating \leq 100$）。

## 输出格式

若存在符合条件的骑手，每行输出一个编号，按排名从高到低；否则输出 `None`。

## 样例

**输入**

```
4
1001 25 95
1002 15 98
1003 30 88
1004 22 92
```

**输出**

```
1001
1004
```

**说明**

| 编号 | 订单数 | 好评率 | 是否达标 |
|------|--------|--------|----------|
| 1001 | 25 | 95% | ✓（25≥20，95≥90） |
| 1002 | 15 | 98% | ✗（15<20） |
| 1003 | 30 | 88% | ✗（88<90） |
| 1004 | 22 | 92% | ✓ |

达标骑手按好评率降序：1001（95%）> 1004（92%），输出 `1001` 和 `1004`。

## 提示

- 定义结构体存储骑手信息，用数组存入所有骑手后先筛选，再对筛选结果排序。
- 比较函数：好评率不同则按好评率降序；相同则按订单数降序。
- 筛选条件：`orders >= 20 && rating >= 90`。
