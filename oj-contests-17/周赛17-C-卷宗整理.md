# C - 卷宗整理

**难度**：★★★☆☆  
**考点**：结构体 + 条件筛选 + 排序  
**GESP 级别**：3级

---

## 题目背景

古堡图书馆存放着大量卷宗，每份卷宗都有案件编号、负责人姓名和密级。探险者只能查阅密级为 A 级的卷宗，并需要按案件编号从小到大整理目录。

## 题目描述

给定 $n$ 份卷宗，每份卷宗包含案件编号（整数）、负责人姓名和密级（字符串）。

筛选出**密级为 `A`** 的卷宗，按**案件编号从小到大**排序，输出负责人姓名（每行一个）。

若没有密级为 `A` 的卷宗，输出 `none`。

## 输入格式

第一行一个正整数 $n$（$1 \le n \le 100$）。

接下来 $n$ 行，每行格式为：

```
案件编号 负责人姓名 密级
```

案件编号为正整数（$1 \le$ 编号 $\le 10000$），姓名为不含空格的字符串（长度 $\le 20$），密级为单个大写字母（A/B/C 之一）。

## 输出格式

按案件编号升序，每行输出一个负责人姓名。

## 样例

**输入**

```
5
302 Alice A
105 Bob B
201 Carol A
408 Dave C
103 Eve A
```

**输出**

```
Eve
Carol
Alice
```

**说明**：密级 A 的卷宗编号为 103(Eve)、201(Carol)、302(Alice)，按编号升序排列。

```cpp
struct Case {

    return a.id < b.id;
```
